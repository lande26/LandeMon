## What is Landemon?

Landemon is a distributed streaming platform to browse and stream **500,000+ movies and TV shows** powered by the TMDb API. It's built as a side project вҖ” but engineered like it isn't.

The stack is designed around real distributed systems principles: edge caching, serverless connection pooling, **WebRTC Screen Sharing (Theater Mode)** for perfect synchronization, and a recursive **Cloudflare Ad-Proxy** for a clean, uninterrupted viewing experience. Everything runs at **$0/month**.

---

## Features

- рҹҺ¬ &nbsp;**Browse & Stream** вҖ” 500,000+ movies and TV shows via TMDb API with ratings, cast, and recommendations.
- рҹ”җ &nbsp;**Premium Auth Guarding** вҖ” Seamless sign-in popups for guest users trying to access Watch Party, Bookmarks, or History.
- рҹ”– &nbsp;**Bookmarks & Watch History** вҖ” Synced to Neon Postgres, accessible on any device.
- рҹҺү &nbsp;**Screen-Shared Watch Party** вҖ” Perfect synchronization via **WebRTC Screen Sharing** and a dedicated **Theater Mode** to prevent recursion.
- рҹ’¬ &nbsp;**Real-time Chat** вҖ” Built into Watch Party via LiveKit Data Channels.
- рҹӣЎпёҸ &nbsp;**Cloudflare Ad-Proxy** вҖ” Recursive edge worker that strips malicious ads and popups from streaming providers.
- вҡЎ &nbsp;**Edge-cached responses** вҖ” TMDb API calls cached in Upstash Redis, sub-10ms cache hits.
- рҹ”Ғ &nbsp;**Stream Fallback Chain** вҖ” 9+ providers with circuit breaker, never breaks mid-watch.
- рҹҢҷ &nbsp;**Dark Theme** вҖ” Clean, aesthetic UI with Framer Motion animations and responsive layout.

### Key Design Decisions

| Decision | Why |
|---|---|
| WebRTC Screen Sharing | Replaces fragile sync models. Host broadcasts their screen directly to guests for frame-perfect sync. |
| Cloudflare Ad-Proxy | A recursive worker that intercepts 3rd-party frames, stripping ads and popups before they reach the user. |
| Theater Mode | A minimal, dedicated window for the Host to share, preventing the "infinity mirror" effect during screen sharing. |
| Upstash Redis for caching | Serverless, pay-per-request, globally replicated вҖ” no idle cost. |
| 9-provider fallback chain | Any single provider can go down вҖ” the chain ensures playback never breaks. |
